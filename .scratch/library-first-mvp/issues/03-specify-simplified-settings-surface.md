# Specify the simplified settings surface

## Question

Turn the settled settings intent into a precise user/admin settings contract.

All users retain theme, display name, Kindle address, notification email address, and notification enablement. Admins, not users, control usernames, passwords, and instance configuration. Decide the page structure and API/settings ownership, which existing Delivery Preferences/output mode/destination/metadata-provider controls are removed versus retained as admin-only configuration, and the migration/deletion approach for obsolete user settings. Notification event selection and delivery behavior remain out of scope.
